# 요구사항 정의

## 최종 프로젝트 목표

LTE Data망을 이용해 원거리 방송 현장에서 사용할 수 있는 헤드셋 기반 인터컴 앱과 PC 서버 프로그램을 만든다.

## 사용자

- 방송 현장 스태프
- 원격 카메라/중계/진행 인력
- PC에서 전체 인터컴 상태를 관리하는 운영자

## 핵심 사용 시나리오

1. 운영자가 PC 서버 프로그램을 실행한다.
2. 서버에서 인터컴 룸을 만들고 룸 코드를 공유한다.
3. Android 사용자가 앱에서 룸 코드로 접속한다.
4. 사용자는 헤드셋으로 듣고 Push-to-Talk 또는 상시 송신으로 말한다.
5. 네트워크가 불안정해도 앱은 재연결하고 현재 상태를 명확히 표시한다.

## 기능 요구사항

### Android 앱

- 룸 코드 입력 후 서버 접속
- 참가자 목록 표시
- 내 마이크 상태 표시
- Push-to-Talk 모드
- mute/unmute
- 유선 헤드셋, USB-C 오디오, 블루투스 헤드셋 지원
- 백그라운드에서도 통화 유지
- 네트워크 품질 표시
- 재연결 상태 표시

### PC 서버 프로그램

- 서버 시작/종료
- 룸 생성/삭제
- 룸별 참가자 목록 확인
- 참가자별 연결 상태 확인
- signaling 처리
- 음성 중계 또는 믹싱 처리
- 로그 확인

## 비기능 요구사항

- 저지연: 일반 LTE 환경에서 체감 지연을 가능한 낮게 유지
- 안정성: LTE 품질 저하, IP 변경, 일시 단절에 대응
- 보안: 룸 코드 또는 토큰 기반 참가 제한
- 확장성: 추후 iOS 앱이 같은 서버에 접속 가능해야 함
- 운영성: 현장에서 PC 서버 상태를 쉽게 확인 가능해야 함

## MVP 제외 항목

- 영상 통화
- 녹음/아카이브
- 복잡한 사용자 계정 관리
- iOS 앱 구현
- 클라우드 배포
- 엔터프라이즈 권한 관리
